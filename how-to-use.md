# Инструкция по запуску

Проект представляет собой Go-сервис. Ниже — минимальный порядок запуска, если у вас уже есть распакованная папка с исходниками.

## 1. Откройте терминал в корне проекта

Перейдите в папку, где лежат исходные файлы проекта, и выполняйте команды из этого каталога.

## 2. Проверьте зависимости

Убедитесь, что установлены:

- Go;
- PostgreSQL;
- Docker, если база запускается в контейнере.

## 3. Настройте переменные окружения

Сервис читает конфигурацию из окружения. Минимально нужны:

- `DATABASE_URL`
- `HTTP_HOST`
- `HTTP_PORT`
- `SHUTDOWN_TIMEOUT_SEC`

По-сути это все прописано по умолчанию.

Для работы AI-модуля также нужны:

- `YANDEX_API_KEY`
- `YANDEX_FOLDER_ID`
- `YANDEX_PROMPT_ID`

Они находятся в key.md и их нужно перенести в соответствующие переменные в ./rules/engine.go.

## 4. Поднимите PostgreSQL

1. Запустить docker;
2. Прописать в cmd: docker-compose up;
3. Подождать завершения создания БД;

- Есть питон скрипт который создает набор тестовых данных для БД. Если хочется свой набор данных, то надо настроить конфиг в файле, запустить и получившийся seed_data.sql закинуть в init-scripts на место 02_seed_data.sql.
- Можно проверить базу данных, используя docker exec -it wb_marketplace_postgres psql -U postgres -d wb_marketplace и SQL запросы к БД (ну или базовый \dt).

## 5. Запустите сервис

Из корня проекта выполните:

```bash
go run ./cmd/server/main.go
```

## 6. Проверьте запуск

После старта сервер должен быть доступен на адресе из `HTTP_HOST` и `HTTP_PORT`.

Например:

```text
http://localhost:8080/health
```
для проверки статуса сервера.

## 7. Проверка анализа

Для ручного запуска анализа используйте соответствующий endpoint административного API, если он подключен в проекте.

## 8. Запуск тестов

Чтобы проверить проект, выполните:

```bash
go test ./...
```

Если нужно запустить только тесты сервиса анализа:

```bash
go test ./... -run TestRunManualAnalysisCreatesRecommendations -v -count=1
```

P.S. флаг -count=1 лучше часто не использовать, он отключает кэширование из за чего каждая итерация тестирования отправляет новый запрос в ИИ (баланс в облаке небольшой).
В будущем рассматривается вариант перехода на локальную ИИ модель.

## Примечание

Если AI-модуль не настроен, сервис может использовать fallback-логику, но для полноценной работы рекомендаций лучше задать все переменные Yandex Cloud.
